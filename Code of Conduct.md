# Code of Conduct for Group 28 Project

## Software Engineering Methods
   
   
### In Regards to Meetings, Collaboration, and Goals:
   
1. Meetings will be held at regular intervals throughout the project, with a minimum of one a week.
2. Meeting times will be discussed on the group Discord server or in person, and the finalised date and time posted in the Discord server.
3. Failure to attend/regular late attendence to meetings will be penalised.
4. All work will be distributed fairly among the group members.
5. If required, members may request to swap tasks with another member or may agree to complete extra tasks for them, but only if another member is willing.
6. All work will be completed in a timely and professional manner, and if extensions are required, these will be requested a maximum of twenty-four hours before the deadline agreed, along with the reason for the delay.
   
### In Regards to Plagarism, Workplace Ethics, and Safety:
   
1. Each member is responsible for their own work and shall strive to credit any external sources used during this project for information or solutions.
2. Each member will strive to maintain a professional and ethical enviroment within the workplace both physically and in digital communication.
3. Where group members do not feel fairly treated, all members shall strive to understand their complaint and resolve the issue swiftly.
4. If the issue is or cannot be resolved, group members are encouraged to contact their personal tutors or the course lecturer.
   
### In Regards to the GitHub Repository and Kanban Board:
   
1. Each group member shall work on a clone of either the Main repository branch or a Feature branch currently in-development.
2. All members agree to never edit the Main repository branch except through merging fully tested Feature branches and with the consent of the entire group.
3. All members agree to regularly create and push commits to GitHub to maintain a clear documentation of the process and increases recoverability in the event of an error.
4. All members agree to maintain the Kanban board (when implementated) and to take resposibility for wasted time and effort should they fail to do so.
   
### In Regards to Updating the Code of Conduct:
   
1. This Code of Conduct was created on the 10/10/2024 and may be updated on the request of any group member.
2. All changes must be approved by all group members in the interests of fairness.
