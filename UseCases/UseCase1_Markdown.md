SEM Group 28: Use Case \#1

| **USE CASE \#1**               | Generate Population Report                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
|--------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **Goal in Context**            | User selects type and scope for report, then a report is generated according to these criteria for the user to view.                                                                                                                                                                                                                                                                                                                                                                                                    |
| **Scope & Level**              | Program, Primary task                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| **Preconditions**              | User has a working computer, the program and its sup-components downloaded.                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| **Success End Condition**      | A report showing the data stored in the program’s database, organised by the ‘type’ and ‘scope’ chosen by the user is generated and displayed on-screen to the user.                                                                                                                                                                                                                                                                                                                                                    |
| **Failed End Condition**       | The program goes not generate a report, the program generates an erroneous or incomplete report, the program does not display anything to the user.                                                                                                                                                                                                                                                                                                                                                                     |
| **Primary, Secondary Actors**  | User, any human or computer that directly interacts with the program. The program, the application used to generate reports.                                                                                                                                                                                                                                                                                                                                                                                            |
| **Trigger**                    | User runs application.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| **DESCRIPTION**                |**Action**   |                                                                                                                                                                                                                                                                                                                                     
| **Step No.1**                  |User starts up the program.   |                                                                                                                                                                                                                                                                                                                   
| **Step No.2**                  |The program displays to user the list of possible types for organising the report.     |                                                                                                                                                                                                                                                          
| **Step No.3**                  |The user chooses one of the options.                                                    |                                                                                                                                                                                                                                                         
| **Step No.4**                  |The program displays to user the list of possible scopes for the report.                 |                                                                                                                                                                                                                                                        
| **Step No.5**                  |The user chooses one of the options.                                                      |                                                                                                                                                                                                                                                       
| **Step No.6**                  |The program asks the user to input a number for the range or ‘0’ for all.                  |                                                                                                                                                                                                                                                      
| **Step No.7**                  |The user enters a number.                                                                   |                                                                                                                                                                                                                                                     
| **Step No.8**                  |The program retrieves data from its database.                                                |                                                                                                                                                                                                                                                    
| **Step No.9**                  |The program organises the data into a list according to the criteria chosen by the user.      |                                                                                                                                                                                                                                                   
| **Step No.10**                 |The program displays the list to the screen in a suitable report format.                       |                                                                                                                                                                                                                                                  
| **EXTENSIONS**                 |**Branching Action**                                                                            |                                                                                                                                                                                                                                                 
| **7a**                         |If the user inputs a negative number or invalid character, the program displays an error message to the screen saying it is an invalid value then returns to step 6.     |                                                                                                                                                                                                                                                                                                                                   
| **9a**                         |The program flags that there is no data and instead displays a message saying that no data was found to the screen.                                                       |                                                                                                                                                                                                                                                                                                                                          
| **SUB-VARIATIONS**             |**Branching Action**                                                                                                                                                       |                                                                                                                                                                      
| **1**                          |The ‘type’ options given to the user are:  Countries.  Cities.  Capital Cities.                                                                                             |                                                                                                                                                                                                                                                                                                                                          
| **2**                          |The ‘scope’ options for the ‘type’ Countries given to the user are:  World.  Continent.  Region.                                                                             |                                                           |
| **3**                          |The ‘range’ decides the number of items to display on the report to the user or, for ‘0’, displays all available items.                                                       |
